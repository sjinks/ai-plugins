## RISK-001: Reflected `name` without sanitization or length limit

- Category: security
- Severity: high
- Confidence: high
- Area: routes

Evidence:
- `src/routes.js` reads `name` from the query string and passes it to `buildGreeting`.
- `src/service.js` concatenates `name` into the response with no length cap or encoding.

Why it matters:
- Untrusted input is reflected into the response body; downstream consumers that render it as HTML could be exposed, and there is no bound on response size.

Suggested verification:
- Send `GET /greeting?name=` with a very long value and an HTML payload and observe the response.

Suggested mitigation:
- Cap `name` length and explicitly set/escape the output for the response content type.

Related tests:
- No test currently exercises the HTTP reflection path (GAP-001).
